`/home-b` (`src/app/_home-b/`, noindex draft) is the user's sandbox for redesigning homepage sections. Pattern: keep the original section, stack 3 candidate treatments after/in place of it, pick one later, then promote the winner to the live home (`src/app/_home/`) — which the approved plan calls the fixed baseline.

In progress (as of 2026-07-24):
- **Beyond the website** (`beyond-options/`): kept original + 3 navy variants — Console (segmented tabs→stage), Orbit (radial map), Dossiers (accordion). Awaiting the user's pick.
- **Clear ownership slot (section 10)**: the user decided ownership should NOT be a dedicated section — it's engagement-specific (sometimes MindWP holds ownership) and handled in the meeting. Reframed the slot to "the website isn't a launch, it compounds." Original ownership section removed from home-b; replaced by 3 candidates in `compound-options/` — Stack (build-up), Arc (accumulation sweep), Snowball (core gathers assets). Eyebrow `NOT A LAUNCH — AN ASSET`, heading "A smart website doesn't peak at launch. It compounds." Awaiting the user's pick.

**Why:** the user iterates on live renders and rejects designs that read as comparison/editorial or are too tall; they favour compact, interactive, distinctive sections.

**How to apply:** [[home-plan-fixes-section-10]] — promoting the compounds section to the live home diverges from the approved plan and needs an explicit plan update, not a silent swap.
