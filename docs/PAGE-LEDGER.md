# MindWP page ledger

This ledger owns current page state and reference permission. Its dimensions are independent: approved meaning can exist before a desktop draft; a rendered draft can await the user’s visual decision; visual approval can exist before production finishing or final audit; publication never follows by implication.

## State vocabulary

| Dimension             | Allowed values                                                                                                                                  |
| --------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| **Meaning**           | absent, research, draft, baseline, approved, unclassified                                                                                       |
| **Desktop draft**     | not started, in progress, rendered, unavailable, superseded, rejected, not applicable, unclassified                                             |
| **Visual decision**   | not started, pending, approved, revise, rejected, superseded, not applicable, unclassified                                                      |
| **Production finish** | not started, in progress, complete, not applicable, unclassified                                                                                |
| **Final audit**       | not started, in progress, passed, issues found, not applicable, unclassified                                                                    |
| **Publication**       | unknown, not approved, approved, live, retired, not applicable                                                                                  |
| **Reference role**    | none, unclassified, content-only, geometry reference, visual target, production reference, brand/quality reference, negative evidence, excluded |

Put qualifications and the next useful action in the evidence column. The default workflow is defined in [PAGE-WORKFLOW.md](./PAGE-WORKFLOW.md); these labels record evidence rather than adding another lifecycle.

## Update rule

Read the relevant row during intake. Update a state only after approved meaning, a rendered desktop draft, a user visual decision, verified finishing work, a final audit, an explicit publication instruction, or a deliberate reference-role decision changes what is known. Record exact source, route, capture, commit, or artifact evidence when useful. Never promote a draft, route, reference, audit, or publication state by inference.
